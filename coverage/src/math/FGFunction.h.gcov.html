      54             : /** Represents a mathematical function.
      55             : The FGFunction class is a powerful and versatile resource that allows
      56             : algebraic functions to be defined in a JSBSim configuration file. It is
      57             : similar in concept to MathML (Mathematical Markup Language, www.w3.org/Math/),

      59             : A function definition consists of an operation, a value, a table, or a property
      60             : (which evaluates to a value). The currently supported operations are:
      61             : - sum (takes n args)

     106             : An operation is defined in the configuration file as in the following example:
     107             : 
     108             : @code
     109             :   <sum>
     110             :     <value> 3.14159 </value>
     111             :     <property> velocities/qbar </property>

     114             :       <property> metrics/wingarea </property>
     115             :     </product>
     116             :   </sum>
     117             : @endcode
     118             : 
     119             : A full function definition, such as is used in the aerodynamics section of a
     120             : configuration file includes the function element, and other elements. It should
     121             : be noted that there can be only one non-optional (non-documentation) element -
     122             : that is, one operation element - in the top-level function definition.
     123             : Multiple value and/or property elements cannot be immediate child
     124             : members of the function element. Almost always, the first operation within the
     125             : function element will be a product or sum. For example:
     126             : 
     127             : @code
     128             : <function name="aero/moment/Clr">
     129             :     <description>Roll moment due to yaw rate</description>
     130             :     <product>
     131             :         <property>aero/qbar-area</property>
     132             :         <property>metrics/bw-ft</property>
     133             :         <property>aero/bi2vel</property>
     134             :         <property>velocities/r-aero-rad_sec</property>
     135             :         <table>
     136             :             <independentVar>aero/alpha-rad</independentVar>
     137             :             <tableData>
     138             :                  0.000  0.08
     139             :                  0.094  0.19
     140             :             </tableData>
     141             :         </table>
     142             :     </product>
     143             : </function>
     144             : @endcode
     145             : 
     146             : The "lowest level" in a function is always a value or a property, which cannot
     147             : itself contain another element. As shown, operations can contain values,
     148             : properties, tables, or other operations. In the first above example, the sum
     149             : element contains all three. What is evaluated is written algebraically as:
     150             : 
     151             : @code 3.14159 + qbar + (0.125 * wingarea) @endcode
