      55             : /** Encapsulates a switch for the flight control system.
      56             : 
      57             : The switch component models a switch - either on/off or a multi-choice rotary
      58             : switch. The switch can represent a physical cockpit switch, or can represent a
      59             : logical switch, where several conditions might need to be satisfied before a
      60             : particular state is reached. The value of the switch - the output value for the
      61             : component - is chosen depending on the state of the switch. Each switch is
      62             : comprised of one or more tests. Each test has a value associated with it. The
      63             : first test that evaluates to true will set the output value of the switch
      64             : according to the value parameter belonging to that test. Each test contains one
      65             : or more conditions, which each must be logically related (if there are more than
      66             : one) given the value of the logic attribute, and which takes the form:
      67             : 
      68             :   property conditional property|value
      69             : 
      70             : e.g.
      71             : 
      72             :   qbar ge 21.0
      73             : 
      74             : or,
      75             : 
      76             :   roll_rate == pitch_rate

      78             : Within a test, additional tests can be specified, which allows for
      79             : complex groupings of logical comparisons. Each test contains
      80             : additional conditions, as well as possibly additional tests.
      81             : 
      82             : @code
      83             : <switch name="switch1">

      99             : </switch>

     103             :
